About this Application

Use Instructions

    There are 4 ways to interact with this application on a computer
  • Use your mouse or touch to click the desired button
  • 3 key input
    • back one key: 'a'
    • forward one key: 'd'
    • Select: 's'
  • 5 key input with additional keys corresponding to up and down
    • back one key: 'f'
    • forward one key: 'h'
    • up one key: 't'
    • down one key: 'b'
    • select: 'g'
  • 5 key input with additional keys corresponding to up and down
    • back one key: 'j'
    • forward one key: 'k'
    • up diagonal right one key: 'o'
    • up diagonal left one key: 'i'
    • down diagonal right one key: ','
    • down diagonal left one key: 'm'
    • select: 'k'

The test

I wanted to test the impact adding two keys has on the speed of input. So I tested five equations 5 times using the three custom coded user input keys. The equations used for testing are listed below. When the equation was completely entered I pressed the '=' button and then cleared it with 'C' button before entering the next equation.

    Equations used in experiment
  • (9+5)/3
  • (7*4)+1
  • (8-6)/2
  • 5*5*5
  • (7/3+6-1)*2

Conclusion

The fastest method of input was 5 keys and the slowest was 7 keys. The time it took to use 3 keys was close to the time it took to use 5 keys. I believe that using 5 keys would have been even faster if the keys were directly lined up so that the key corresponding to up was directly above the enter key and same with the down button. The results also demonstrate that more keys is not always better. Having 7 keys is not an intuitive means of input. It is more intuitive to have an up and a down button then diagonal. This is an important consideration to consider when assessing means of input.

Additional Test

I was also curios about how it would be using the key input for various types of input instead of hands. I test the 3 button input method using my nose once and my foot once. Both methods of input were substantially harder than using hands. The hardest thing about using my nose was that every time you then had to look up to make sure you were on the right key and that it selected properly. I was expecting to have more errors as I input the equations, but I managed to have very few errors and never accidentally hit the wrong key. Honestly, this experience was very dizzying and started to give me a headache before I even made it through all of the equations. Thus, if nose were to be used as the primary input method there would need to be a better input design that would not require constantly having to look up and down.

Using my foot as a means of input was faster then using my hand but again was accompanied by challenges. One of the biggest challenges that I encountered was the I would hit the wrong key quite easily. The other main challenge that I encountered was that it required me to hold my foot in a pointed position which became uncomfortable quite quickly. Thus, if I were to design a system where the foot tapped buttons as a means of input I would consider making them larger and position them in a more natural fashion.
